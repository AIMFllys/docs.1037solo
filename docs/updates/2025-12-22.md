## 1037StudyBox 1.0 → 2.0 战略转型（背景说明）

在 v1.0 阶段，1037StudyBox 以"综合学习资源导航"为核心思路，堆叠了大量功能模块，但整体架构混乱、用户体验割裂。经过评估，团队决定：

- 摒弃资源堆叠式设计，转向极简 AI 搜索体验
- 从 Next.js App Router 重新立项，建立可持续的技术底座
- 核心产品理念变更：**一个足够聪明的搜索框，胜过一百个功能按钮**

### 【面向开发者】

#### v2.0.001 🏗️ 工程底座与 SplashScreen
- **框架选型**：Next.js 14+ (App Router) + Tailwind CSS + Framer Motion。
- **目录结构**：建立 `features/` 模块化拆分的 Google 级标准目录树。
- **SplashScreen**：使用 Framer Motion 加载 `blessings.json` 随机祝福语动态开屏。
- **HeroSearchBar**：具备 `layoutId` 平滑上移效果的居中搜索框。

#### v2.0.002 🧭 侧边栏架构修复
- **Mini Mode**：折叠后保持图标居中可见（64px 宽度），修复收起时完全消失的 Bug。
- **功能排序**：确立侧边栏标准顺序（头像 → 新标签 → AI Chat → 打卡 → 收藏 → 论坛 → 音乐 → 旧版 → 反馈）。
- **UserNavMenu**：右上角新增悬浮用户菜单组件。

#### v2.0.003 ⚙️ 设置系统与 Zustand 状态管理
- **设置面板**：`features/settings` 下实现左侧导航 + 右侧内容的 Edge 风格深度设置。
- **状态管理**：引入 Zustand 管理侧边栏显隐（Sidebar Visibility）与全局状态同步。
- **RefreshButton**：右上角带旋转动画的一键刷新组件。

#### v2.0.004 🌗 深色模式与品牌视觉
- **Dark Mode**：原生实现深色/浅色模式切换，设置中立即生效。
- **Logo**：加入"1037 SOLO"品牌 Logo（粗体 1037 + 细体 SOLO）。
- **收藏同步**：侧边栏可实时渲染 Mock 学术资源（Google Scholar 等）。
- **底部微件**：底部 Widget Hover 触发机制，减少初始视觉干扰。
