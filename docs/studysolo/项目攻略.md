# 项目攻略（操作及开发指北）

## 📌 用户手册：如何驾驶这台自动化学习引擎？

1. **确立灯塔目标**：
   在控制面板中央输入你的任何“大而虚”的想法。比如“解析目前开源界所有大模型多模态架构的优劣对比”。
2. **等待系统绘制图纸**：
   引擎会短暂陷入“逻辑运算期”，接着原本空白的画布会犹如神经突触一般瞬间绽放出一套极尽详细的工作流程拆解图。
3. **审阅并动手改装**：
   不要盲目相信 AI！你可以自由删减不满意的废话节点，或者按自己偏好的学习方式在重点环节拉出新的分支线！全权掌控，随心定制。
4. **引擎点火（Run All）**：
   点击主界面的流式运行指令后，一切自动化开始。每一道数据都会按图纸管线自动向各大 AI 平台下发并行任务请求，你的屏幕会呈现壮观的同时书写、生成、归纳的震撼过程。
5. **深度落盘**：
   无论产出多宏伟，它都在云端被切片封装完毕了，你可以将其汇编输出保存用于你的现实论文综述、开发报告中，无需任何人工排版修饰。

## 💻 开发者指南：深入核心熔炉的禁飞区

想要在本项目进行重火力二次研发？请严格对齐以下工程准入规则。

### 1. 魔法般的“精准上下文穿透”
不要再在全局一股脑地进行 `context string` 粗暴合并了！
StudySolo 使用的是空间几何式的关联传输：每个节点的上下文记忆 **物理上只来源于与它连通的线引流下来的数据结构**！引擎的 `context.py` 会自动沿着 Edge 有向边抽取对应的记忆并灌注传递，这就让极复杂的深层结构免去了幻觉污染可能！

### 2. 极简的“3步新增节点插件”范式
目前所有的运行节点均做到了彻底解耦沙盒化，新增玩法极其容易。
- **Step 1 (引擎层搭建)**：进入后端 `nodes/<类型文件夹>/` 建立包含独有独立 `node.py` （负责执行逻辑，继承基础 BaseNode）与 `prompt.md` 任务描述的小室。
- **Step 2 (调度网注册)**：去 `config.yaml` 在 `node_routes` 中将你的新节点接入指定能力的语言模型信道。
- **Step 3 (视觉层通电)**：去前端 `frontend/src/components/business/workflow/nodes/index.ts` 通过匹配键值为它挂载一个渲染视窗，即刻生效！

### 3. 高压电线的代码治理基准线
团队在全栈实施残酷到偏执的代码整洁约束：
- **行数生命线**：没有任何单一组件/类被允许超过 `300` 行！入口调度页更被死锁限制在 `220` 行以下！任何违规新增一旦通过 `pnpm lint:lines` 触网将绝对爆红。
- **绝对的薄片隔离**：前端组件中严厉禁绝夹杂异步网络逻辑和定时轮询，所有脏活只能委托到统一的 Hooks 沉淀与 Services API。确保 UI 是绝对的“薄而透亮的晶片”。
