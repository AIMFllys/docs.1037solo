# StudySolo 项目介绍

## 1. 项目定位
**StudySolo** 是 1037Solo 生态下的核心次时代子项目，定位为**新一代 AI 驱动的可视化学习工作流平台**。如果说 1037Solo 主平台是用户获取信息与进行碎片化对话的入口，那么 StudySolo 则是用户**将长线目标具象化、将复杂学习过程拆解并自动化深度执行**的“第二大脑操作系统”。

它创新性地将大语言模型（LLM）的分析规划能力与节点式（Node-based）可视化画布相结合。用户只需输入一个宽泛的课题目标，系统便会自动规划出包含“分析-分解-检索-结构化生成”等环节的定制化专属工作流，并利用内置的多模型引擎进行流式执行，带来沉浸式、极客范的高效知识链路体验。

## 2. 宏观架构
StudySolo 采用前后端分离的现代化企业级构架，不仅完全继承而且突破了 1037Solo 生态的技术标准：
- **前端 (Next.js 16)**：App Router 架构，搭载 Tailwind CSS v4 与 React 19，以极其严苛的工程规范（单文件 ≤ 300 行）构建。集成 `@xyflow/react` 驱动工作流可视化拓扑，并使用 `Zustand` 负责极低延迟的全局动态画布状态。
- **后端 (FastAPI)**：全异步 ASGI 弹性架构，以独创的 **工作流拓扑编排引擎** 与 **动态节点插件系统** 为核心轴。AI 产生的每个阶段性思考（Token）以高并发无阻塞的 SSE（Server-Sent Events）格式被推送到前端对应节点。
- **数据库底座 (Supabase)**：复用 1037Solo 的 Supabase Project (`hofcaclztjazoytmckup`)，达成 One-ID JWT 联合登录与无缝数据切流，采用严格的行级安全策略（RLS）进行物理级别的租户账单及数据隔离。

## 3. 究竟在解决什么痛点？
1. **学习路径的“重度迷雾”**：面对崭新而宏大的交叉领域课题，传统的聊天式 AI 会陷入“查到深处忘出来”的窘境。StudySolo 抛弃问答轴，直接生成类似心智图的网格化学习链路。
2. **割裂的 AI 工具流水线**：一次深度学习和报告构建，往往需要提取大纲、扩写、跨语种翻译、检索多库资料甚至制作背诵闪卡。这些往往需要不停地复制文本切换窗口，在 StudySolo 内，它们全成了连在同一根管子上的组件节点。
3. **黑箱化不可控的焦虑**：常规 AI 黑盒式地吐出结果一旦跑偏，用户无法半路干预修订上下文。在节点画布上，任何一个出错的上游生成都可以被人类随时暂停并修改修正从而继续传导！
